☁️ Climate Depth — System Design

Temperature layer + multi-day streaks + probabilistic forecast. All changes are additive — nothing existing breaks.

1. Data Model Changes

WeatherDay 3 new fields

type WeatherDay = { event: WeatherEvent // existing minTemp: number // NEW °C maxTemp: number // NEW °C probability: number // NEW 0.0–1.0 streakDay?: number // NEW day 1–7 of streak }

CropType 3 new fields

type CropType = { ...existing fields frostKillTemp: number // NEW °C (e.g. -5) heatStressTemp: number // NEW °C (e.g. 36) droughtTolerance: number // NEW 0–1 }

PlantedCrop 3 new fields

type PlantedCrop = { ...existing fields frostDamage: number // NEW 0–1 droughtStress: number // NEW 0–1 moistureLevel: number // NEW 0–1 }

No breaking changes

All new fields are optional/additive. Existing harvest formula gains two new multipliers. Forecast UI gets temperature display + probability badge. Everything else unchanged.

7. Mitigation — What Protects You

🌡️ Weather forecast

7-day forecast shows streaks coming. Players can early-harvest vulnerable crops before frost hits, or skip planting tender crops in late autumn.

💧 Irrigation machinery

Already in the game. Drip irrigation now has a concrete effect — caps moisture loss during drought. Builds real value for irrigation investment.

🛡️ Crop insurance

Weather insurance already exists. Frost kill and drought now trigger claims automatically. Payout = estimated lost yield × coverage %.

Nothing new to build for mitigation — forecast, irrigation, and insurance are already fully implemented. This design gives them real weight instead of being nice-to-haves.
