Design — Section 2: Store, Engine & UI

How tractor jobs flow through the system + what changes in the UI

⚙️ Engine — New Pure Functions (engine/machinery.ts)

calcJobDays(totalHa, haPerDay)
Returns Math.ceil(totalHa / haPerDay). A 5ha field with a 2ha/day cultivator = 3 days.
getContractorCost(operation, totalHa, prices)
Returns flat fee based on operation × hectares. Transport returns 12% of estimated sale value.
canAssignJob(tractor, attachment, parcels)
Validates: tractor not busy, attachment compatible, parcels in correct state (tilled for plant, etc.).
getTransportCapacity(machines)
Returns total kg capacity from owned truck+trailer combos. Used to check if crop can be self-transported.

🗄️ Store — New State Fields

attachments: OwnedAttachment[]
{ id, typeId } — purchased attachments. Same pattern as OwnedMachine.
tractorJobs: TractorJob[]
Active jobs. Each tractor can only appear once. Completed jobs removed by advanceDay.
trailers: OwnedTrailer[]
{ id, typeId, attachedTo: truckId | null } — tracks which trailer is hitched to which truck.

🔧 Store — New Actions

buyAttachment(typeId) — deduct cost, push to attachments[]
buyTrailer(typeId) — deduct cost, push to trailers[]
hitchTrailer(trailerId, truckId) — sets trailer.attachedTo = truckId
assignJob(tractorId, attachmentId, operation, parcelIds) — validates, computes completesDay, pushes TractorJob
hireContractor(operation, parcelIds) — deduct fee, instantly apply operation (set tilled=true / plant / harvest)
plantCrop updated — now requires parcel.tilled === true, else blocked
harvestCrop updated — now requires owned combine OR blocks with "hire contractor" prompt

📅 advanceDay — Job Processing

Each day, for each active TractorJob:
1. If completesDay <= newDay → apply results (set tilled, plant, spray parcels) → remove job → push day summary entry
2. If still in progress → no action needed (completesDay was pre-calculated at assign time)

Harvest jobs work differently — combine harvests N ha/day, so advanceDay tracks processedHa incrementally until all parcels done.

🖥️ UI Changes

🏪 Market Tab — Split into 2
Current: Machinery + Buildings in one tab
→ Machinery tab: Tractors, Combines, Trucks, Trailers, Attachments
→ Buildings tab: All existing buildings (silo, workshop, greenhouse, etc.)
🚜 Machinery Tab — Redesign
Internal sub-tabs: Fleet · Attachments · Jobs
Fleet: owned tractors/combines/trucks with active job status
Attachments: owned + shop
Jobs: active jobs with progress + assign new job UI
🌾 Fields Tab — Operation Buttons
Each parcel card gets operation buttons based on its state:
Unowned → Buy
Owned + untilled → Till (assign tractor or hire contractor)
Tilled → Plant
Growing → Spray (optional)
Ready → Harvest
🤝 Contractor Modal
When player clicks an operation without the machine:
→ Modal: "You don't own a [machine]. Hire a contractor for $X?"
→ Shows cost breakdown (ha × rate)
→ Confirm → instant completion

New state fields (attachments, tractorJobs, trailers) require a save key bump: v6 → v7. Old saves wiped.
